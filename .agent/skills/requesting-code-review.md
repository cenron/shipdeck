# Requesting Code Review

## Overview

Prepare the work so another engineer can review it efficiently and focus on correctness, regressions, and missing coverage.

## Rules

- Summarize what changed and why.
- Highlight risky areas and unresolved questions.
- Point to the verification you ran.
- Ask for a review before merging or shipping major changes.

## Common Mistakes

- Asking for review without verification.
- Hiding uncertain areas.
- Treating review as a formality.
