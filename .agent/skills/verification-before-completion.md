# Verification Before Completion

## Overview

Do not declare success without evidence. Run the relevant checks and confirm the result before saying the work is done.

## Rules

- Run the smallest relevant verification first.
- Expand to broader checks before merge or handoff.
- Use real output, not assumptions.
- Do not claim completion until the evidence supports it.

## Common Mistakes

- Saying it is done before running the checks.
- Running irrelevant tests and calling it verification.
- Ignoring failing output.
