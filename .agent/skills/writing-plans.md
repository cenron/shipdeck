# Writing Plans

## Overview

Turn a design into a concrete implementation plan with clear steps, file targets, and verification checkpoints.

## Rules

- Break work into small, ordered tasks.
- List files and responsibilities first.
- Prefer step-by-step execution over vague milestones.
- Include verification points for each major task.
- Keep the plan implementation-ready but code-free.

## Common Mistakes

- Writing a summary instead of a plan.
- Skipping file boundaries.
- Mixing design discussion with implementation details.
